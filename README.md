# Ringtone Maker

Trim a song you already own into a `.m4r` ringtone and get clear, illustrated
instructions for installing it on your iPhone. iOS only, SwiftUI, no backend,
no third-party dependencies — system frameworks only (SwiftUI, AVFoundation,
MediaPlayer).

Built to replace "Ringtones – Unlimited Ringtones Maker": same core job
(trim + export), fixing its two biggest complaints — a stale bundled song
library, and no clear way to make a ringtone from your own music.

## Requirements

- Xcode 15+ (targets iOS 17.0+)
- A physical iPhone for testing trim-handle precision, preview playback, and
  the actual install flow (GarageBand/Finder) — the simulator can't validate
  any of these end-to-end.

## Getting started

```
open RingtoneMaker.xcodeproj
```

Build and run on a device. `NSAppleMusicUsageDescription` is already set in
`RingtoneMaker/Info.plist`; Xcode will prompt for media-library access on
first launch.

> This project was scaffolded on a machine without a full Xcode install (CLI
> tools only), so `project.pbxproj` was hand-written rather than generated by
> Xcode. It's a standard single-target app project and should open normally,
> but if Xcode complains about the project file on first open, the fix is
> usually to let Xcode re-save it (File > nothing needed, just build once).

## Project structure

```
RingtoneMaker/
  RingtoneMakerApp.swift          // @main entry point
  ContentView.swift                // Root navigation: Picker -> Trim -> Export -> Instructions
  Models/
    TrimSelection.swift            // Start/end time, fade toggle, 30s clamping logic
  Views/
    SongPickerView.swift           // Wraps MPMediaPickerController, filters non-exportable tracks
    TrimEditorView.swift           // Waveform + drag handles + preview playback + export
    ExportResultView.swift         // Success screen + share sheet
    InstallInstructionsView.swift  // GarageBand (primary) + Finder (alternative) install guide
  Engine/
    WaveformGenerator.swift        // Reads audio samples via AVAssetReader, downsamples to ~300 points
    RingtoneExporter.swift         // AVAssetExportSession + fade AVMutableAudioMix, .m4a -> .m4r
  Resources/
    Assets.xcassets                // App icon (placeholder), accent color
  Info.plist
```

## How it works

1. **Pick a song** — `MPMediaPickerController` (UIKit, wrapped via
   `UIViewControllerRepresentable`). Cloud/DRM-protected tracks
   (`MPMediaItem.assetURL == nil` or `isCloudItem == true`) are rejected with
   a friendly message rather than a silent failure — see
   `MediaExportability` in `SongPickerView.swift`.
2. **Trim** — waveform is generated by reading the audio track with
   `AVAssetReader` and downsampling to ~300 peak points (see
   `WaveformGenerator`), regardless of song length. Two drag handles select a
   range clamped to 30 seconds (`TrimSelection.clamped`). "Preview" plays just
   that range via `AVAudioPlayer`, stopped by a `Timer` sized to the
   selection's duration.
3. **Export** — `RingtoneExporter.export` runs an `AVAssetExportSession`
   (`AVAssetExportPresetAppleM4A`) over the trim's `CMTimeRange`. If fade is
   on, a `AVMutableAudioMix` with two `setVolumeRamp` calls (1.5s in, 1.5s
   out) is attached before exporting. The session outputs `.m4a`; the file is
   then renamed to `.m4r` (same container, just the extension iOS looks for).
4. **Install instructions** — since there's no public API to install a
   ringtone directly, this screen is the actual product. GarageBand's
   "Share Song > Ringtone" is the primary path (works with no Mac); Finder's
   drag-to-Tones is offered as an alternative for Mac users.

## Known constraints (by design, not oversights)

- **30-second cap.** `.m4r` technically supports up to ~40s; 30s was picked
  as the simpler, more conventional story to put in front of users.
- **No streaming-library support.** Only locally downloaded, non-DRM tracks
  can be trimmed/exported — this is an `AVAssetExportSession` limitation, not
  a policy choice, and is surfaced clearly in the picker rather than failing
  silently mid-export.
- **No configurable fade duration.** Fixed 1.5s in/out; adding a slider here
  is explicitly out of scope for v1.

## Before submitting to the App Store

- [ ] Real app icon (currently a placeholder `AppIcon.appiconset` entry with
      no image assets — Xcode will complain until a 1024x1024 icon is added)
- [ ] Test on a physical device: DRM/streaming filter, handle drag precision,
      preview stop-at-boundary accuracy, fade audible in the *exported* file
      (not just preview), and the full GarageBand install path end-to-end
      with no Mac involved
- [ ] Test the Finder/Mac install path as a secondary check
- [ ] App Store Connect: Privacy section (media library access only, "Data
      Not Collected" for everything else), $2.99 price tier, and a visible
      IP-use disclaimer (in-app or in the listing) stating the app is for
      music the user owns or has rights to use as a personal ringtone
- [ ] Screenshots: waveform trim screen + install-instructions cards (the
      trust-builder for buyers wary of bad ringtone apps)
